Atributos globales

Los atributos globales pueden aplicarse a la mayoría de las etiquetas HTML, excepto aquellas que pertenecen únicamente a la estructura del documento como <html>, <head> y <title>. Estos atributos permiten agregar información, comportamiento o estilo a cualquier elemento. Su uso facilita la organización del código y la manipulación mediante JavaScript o CSS.

1.1 Atributos globales más utilizados

id: Identificador único para un elemento. Solo puede existir un id con el mismo nombre en toda la página. Permite apuntar directamente al elemento desde CSS usando "#" o desde JavaScript con document.getElementById(). Es fundamental para manipular contenido de forma precisa.

class: Permite agrupar elementos bajo un mismo nombre de clase. A diferencia del id, varios elementos pueden compartir la misma clase. Es ampliamente utilizado en CSS con el selector "." y en JavaScript con querySelectorAll(). Facilita aplicar estilos o comportamientos en conjunto.



Atributos del formulario (form)

Los atributos del formulario controlan cómo se envían y procesan los datos ingresados por el usuario. Estas configuraciones son esenciales para conectar HTML con aplicaciones, servidores o bases de datos. Los atributos más comunes son “action” y “method”, los cuales determinan a dónde irán los datos y la forma de enviarlos. Estos atributos permiten crear formularios funcionales y organizados.

2.1 action

El atributo action especifica la dirección o archivo donde se enviarán los datos del formulario. Si está vacío, la información se envía a la misma página actual. Su función es conectar el formulario con un servidor o script que procese la información enviada por el usuario.

Ejemplo:
<form action="procesar.php">

2.2 method

El atributo method define cómo se envían los datos al servidor. Existen dos métodos principales: GET y POST. GET envía los datos visibles en la barra de dirección; es útil para búsquedas. POST envía la información de forma oculta; se utiliza para datos privados o registros. Es esencial elegir el método adecuado según el tipo de formulario.

Ejemplo:
<form method="post">



Atributos del input

Los atributos del input permiten controlar cómo se comportan los campos del formulario. Cada atributo añade características importantes como tipo de dato, texto guía, validación o restricciones. Estos atributos garantizan que el usuario introduzca la información correctamente y que los formularios sean más intuitivos. Son indispensables en cualquier formulario moderno.

type: Define el tipo de entrada que se usará como texto, número, correo, contraseña, fecha, archivo, etc. Este atributo determina qué tipo de dato se espera recibir e influye en la interfaz del campo. Ejemplo: <input type="email">

placeholder: Muestra un texto guía dentro del campo antes de escribir. Sirve como instrucción visual para explicar lo que debe ingresarse. Se borra automáticamente cuando el usuario comienza a escribir. Ejemplo: <input placeholder="Escribe tu nombre">

required: Indica que el campo debe ser completado obligatoriamente antes de enviar el formulario. Es una forma simple de validar datos sin usar JavaScript. Si no se llena, el navegador mostrará un mensaje. Ejemplo: <input required>

name: Asigna un nombre al campo, lo cual es necesario para que el servidor identifique el dato enviado. Sin este atributo, un formulario no puede enviar correctamente esa información. Es clave en bases de datos. Ejemplo: <input name="usuario">
